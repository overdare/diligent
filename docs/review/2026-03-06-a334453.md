### 2. Image Upload — Well-Architected Cross-Package Feature
**Impact**: Positive — major feature landed within existing boundaries

The image upload system (2e483f4, dfd68d8, 2e4e0c4, a334453) adds vision model support across 4 commits:

The 4-commit progression (add → serve canonical → drop preview URLs → paste support) shows iterative refinement without throwaway work. Each commit builds on the previous.

### 5. `as` Casts in rpc-bridge.ts Post-Dispatch — Structural Pattern
**Impact**: Low — internal response shape assertions, not security boundaries

9 `as` casts remain in `rpc-bridge.ts:446-710`, all in the post-dispatch section where rpc-bridge extracts data from `DiligentAppServer.handleRequest()` responses to update session state:

| Line | Cast | Purpose |
|------|------|---------|
| 446 | `response.result as { threadId?: string }` | Track new thread ID after THREAD_START |
| 456 | `response.result as { found: boolean; threadId?: string }` | Track resumed thread after THREAD_RESUME |
| 465 | `response.result as { currentEffort?: ThinkingEffort }` | Sync effort on THREAD_READ |
| 481 | `response.result as { mode?: Mode }` | Sync mode on MODE_SET |
| 488 | `response.result as { effort?: ThinkingEffort }` | Sync effort on EFFORT_SET |
| 495 | `response.result as { deleted?: boolean }` | Check deletion on THREAD_DELETE |
| 496 | `parsed.params as { threadId?: string }` | Get deleted threadId |
| 651 | `notification.params as { threadId?: string }` | Route notification by threadId |
| 710 | `notification.params as { item?: { type?: string } }` | Detect userMessage items |

These casts exist because `handleRequest()` returns `JSONRPCResponse` (generic `result?: unknown`), not method-specific typed responses. The protocol already defines typed response schemas (`DiligentClientResponseSchema`) — a typed response helper could narrow `result` based on the request method, eliminating all 9 casts. This is a quality-of-life improvement, not a safety issue — the shapes are guaranteed by `DiligentAppServer`.

**Fix cost**: ~30 minutes — add a typed response extraction utility that narrows by method.

### 14. Provider Registry — Still If/Else Chain
**Horizon**: Adding 4th provider

```typescript
if (provider === "openai") {
  stream = createOpenAIStream(apiKey, this.baseUrls.openai);
```
