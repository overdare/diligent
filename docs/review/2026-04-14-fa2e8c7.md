## Forward Compatibility

### Web Reducer Adoption Is the Final Convergence Step
**Horizon**: Blocks full reducer convergence

With collab events added and test suite in place, the protocol reducer covers the base live-streaming state needed by all clients. Web adoption requires:
1. Replace Web's `reduceAgentEvent()` with `applyAgentEvents()` for base state
2. Keep Web-specific extensions (RenderItem mapping, thinking-duration, content-block finalization) as a thin layer
3. Ensure collab-reducer.ts and tool-reducer.ts can compose with the shared base

This is a manageable refactor. Web's `thread-store.ts` (648 LOC) will not shrink dramatically because item-level state management is intentionally client-specific, but the event-type switch duplication will be eliminated.

## Novel Perspective

### The Conversation Reducer Test Suite Tests Today's Behavior, Not the Convergence Contract

- **Why no one has seen this**: Previous reviews correctly identified the need for a test suite (2026-04-13 review, Novel Perspective). Now that the test suite exists (344 LOC, 13 describe blocks), the natural assumption is the risk is mitigated. But examining what the tests actually cover reveals a subtler gap.
- **Evidence**: `packages/protocol/test/conversation-reducer.test.ts` tests `applyAgentEvents()` against the 19 event types it currently handles. It verifies the protocol reducer's own behavior in isolation. However, the convergence goal requires Web and CLI to delegate their base state to this reducer. Neither Web's `thread-store.test.ts` nor CLI's tests exercise the shared reducer — they test their own independent implementations. When Web adopts the shared reducer, its existing test suite will continue testing Web-specific logic but will not verify that the shared reducer produces identical base-state transitions to what Web's independent reducer currently produces.
- **Implication**: Before Web adoption, add a cross-validation test: feed the same event sequences to both Web's current `reduceAgentEvent()` and protocol's `applyAgentEvents()`, and assert that the overlapping `ConversationLiveState` fields produce identical results. Any divergence reveals either a protocol reducer bug or an undocumented Web-specific behavior that must be preserved as a client extension rather than eliminated. This test would be temporary (deleted after Web adopts) but prevents subtle state-behavior regressions during the migration.
